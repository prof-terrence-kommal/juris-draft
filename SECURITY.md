# Security Policy

## Supported versions

Juris Draft is currently in early public development. Security fixes will be applied to the latest released version.

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |

## Security scope

Security review for this project includes:

- CLI code and local file handling;
- dependency hygiene;
- GitHub Actions workflow permissions;
- repository automation;
- examples, fixtures, and generated outputs;
- protection against accidental publication of sensitive or confidential material.

## Confidentiality and legal information

Juris Draft does not require users to submit confidential, privileged, or personal legal information.

Do not include real client files, privileged facts, confidential legal documents, private personal information, or sensitive business information in:

- GitHub issues;
- pull requests;
- example files;
- public test fixtures;
- public generated outputs.

Use fictional or anonymised examples only.

## Reporting a vulnerability

If you believe you have found a security issue, please open a private security advisory on GitHub where available, or contact the maintainer directly through the public profile links.

Please include:

- a description of the issue;
- steps to reproduce;
- affected files or commands;
- potential impact;
- suggested mitigation, if known.

## Responsible use

Juris Draft is not legal advice and does not replace review by a qualified legal professional. The project is designed to support structured first drafts, drafting packets, and workflow infrastructure with human review.
